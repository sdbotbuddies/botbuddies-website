🚀 Student Setup Guide

Getting Started

Follow these steps to set up your coding environment. You only need to do this once — then you're ready to code all year!

1
// install

Install Git

Git is how your code travels between your computer and GitHub. Pick your operating system:

Open the Terminal app and run:
xcode-select --installclick to copy
A popup will appear — click Install and wait for it to finish.
Download and install Git for Windows:
https://git-scm.com/download/winclick to copy
Use all the default settings during installation.
💡 Not sure if you already have it? Open Terminal (Mac) or Command Prompt (Windows) and type git --version. If you see a version number, you're good!
2
// install

Install uv — Python Manager

uv installs Python and all the tools you need automatically. It's way easier than installing Python separately.

Open Terminal and run:
curl -LsSf https://astral.sh/uv/install.sh | shclick to copy
When it finishes, close and reopen Terminal.
Open PowerShell and run:
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"click to copy
When it finishes, close and reopen PowerShell.
💡 To verify it worked: type uv --version in a new terminal window. You should see a version number.
3
// install

Install VS Code + Python Extension

VS Code is your coding home base. It's free and used by developers at Microsoft, Google, and NASA.

  • Download VS Code from code.visualstudio.com and install it
  • Open VS Code, click the Extensions icon (four squares) in the left sidebar
  • Search for Python and install the one made by Microsoft
💡 That's it for VS Code! The Python extension gives you test running, autocomplete, and error highlighting — all the good stuff.
4
// setup

Clone Your Repo in VS Code

Your coach will send you a personal repo link — this is your coding workspace for the whole year. Here's how to open it:

  • Open VS Code
  • Press Ctrl+Shift+P (Windows) or Cmd+Shift+P (Mac)
  • Type Git: Clone and press Enter
  • Paste the repo link your coach sent you and press Enter
  • Choose a folder on your computer to save it, then click Open
💡 Your repo link looks like: https://github.com/sdbotbuddies/student-yourname
5
// setup

Tell Git Who You Are

Git needs to know your name and email so your commits are tagged correctly. Open the integrated terminal in VS Code with Ctrl+` (same on Windows and Mac) and run these two commands — replace the name and email with what your coach told you to use:

In the Terminal inside VS Code, run:
git config --local user.name "Your Name"click to copy
git config --local user.email "your@email.com"click to copy
In the PowerShell terminal inside VS Code, run:
git config --local user.name "Your Name"click to copy
git config --local user.email "your@email.com"click to copy
💡 You only do this once per computer. The --local flag sets it just for this repo, so it won't affect anything else on your machine.
6
// setup

Allow Automatic Setup

When VS Code opens your repo for the first time, it will show a notification at the bottom of the screen:

"This workspace has tasks configured to run automatically."
→ Click Allow

This runs uv sync in the background, which installs Python 3.11 and pytest automatically. It takes about 30 seconds.

💡 You only see this once. After clicking Allow, your environment is set up forever on this computer.
7
// verify

Run Your First Test

Let's make sure everything is working. Look for the flask icon ⚗️ in the left sidebar of VS Code — that's the Test Explorer.

  • Click the ⚗️ flask icon in the left sidebar
  • You should see your first assignment listed: a01-python-basics
  • Click the ▷ Run All Tests button at the top
  • You'll see red ❌ marks — that's normal! It means the tests are working and waiting for your code.
💡 Red is good here! It means the tests found your empty solution. Your job is to write code in solution.py until they all turn green ✅.
8
// workflow

Submit Your Work

Once your tests go green, here's how to submit using VS Code:

  • Click the Source Control icon 𝌡 in the left sidebar
  • Click + next to Changes to stage your files
  • Type a short message in the box — like "solved a01"
  • Click the dropdown arrow next to Commit and choose Commit & Push

Within seconds, GitHub automatically runs all the tests — including some hidden ones your coach added. Check the Actions tab on GitHub to see your full results!

💡 Hidden tests? Yep — some tests are secret. That's part of the challenge. If your visible tests all pass but a hidden one fails, you've found an edge case to fix. Welcome to real software development! 🎉
🎉

You're All Set!

Your environment is ready. Open your first assignment at assignments/a01-python-basics/README.md, read the instructions, and start coding. You've got this!

// troubleshooting

Something not working?

Common issues and how to fix them.

!
// issue

Tests not showing in the Testing panel

VS Code is using the wrong Python — the system Python instead of the one installed by uv sync.

  • Press Ctrl+Shift+P → type Python: Select Interpreter → Enter
  • Choose the one that says .venv/bin/python (Mac/Linux), .venv\Scripts\python (Windows), or 'botbuddies': uv
  • Click the refresh icon in the Testing panel — tests should appear
💡 Can't find .venv in the list? Click Enter interpreter path... → Find... and navigate to your repo folder → .venv → bin → python (Mac/Linux) or .venv → Scripts → python (Windows)
!
// issue

"No module named pytest" error

The setup task ran but VS Code is still using the wrong Python interpreter.

  • Check that .venv folder exists inside your repo folder
  • If it doesn't exist, open the terminal (`` Ctrl+` ``) and run:
    uv syncclick to copy
  • Then follow the steps above to select the .venv interpreter
💡 Quick check: Look at the bottom status bar in VS Code. It shows the active Python version. It should say 3.11 — if it shows 3.13 or another version, the wrong interpreter is selected.
!
// issue

The setup task didn't run automatically

VS Code didn't prompt "Allow automatic tasks?" or you accidentally clicked Dismiss.

  • Press Ctrl+Shift+P → type Tasks: Run Task → Enter
  • Select Setup: Install dependencies
  • A terminal will open and run uv sync — wait for it to finish
💡 Still not working? Ask your coach — they can help you get set up in a few minutes.
